# 经纬书房 · 个人知识图书馆

一个零依赖、可离线运行的中外人文社科双轨时间轴。视觉与交互承接旧摄影网站“光影集”的深色画廊、分类、详情与收藏逻辑。

## 当前内容

- 六科：文学、社会学、政治学、历史学、哲学、法学
- 两个维度：中国 / 全球
- 5 个时代章节：时代背景、文风/方法与关键转向
- **109 位代表人物**（首批 24 位核心 + 85 位扩展人物），含生平、时代坐标、文风/论证方式与核心思想
- **176 部代表作品**及内容简介
- 六科思想脉络，可沿继承、改写或解释性对照继续阅读
- 人物详情下钻与“时代图谱 / 中外对照”双模式
- **38 组**中外对照节点，覆盖先秦至数字时代
- **42 本**起步书目
- 时间、时代、学科与全文搜索筛选
- **核心 / 扩展人物**筛选切换（"仅展示首批 24 位核心人物"）
- 时间节点收藏
- 书籍“想读 / 在读 / 读完”进度，保存在浏览器 `localStorage`
- 桌面与手机响应式布局

## 运行

推荐双击 `启动.command`，它会打开浏览器并启动带进度保存功能的本地服务。也可以在本目录运行：

```bash
python3 server.py
```

然后访问 `http://127.0.0.1:8877`。直接双击 `index.html` 也能浏览，收藏与进度会退回浏览器本地存储。

## 数据维护

所有时代、人物、作品、关系、时间节点与书目都在 `data.js`。`ERA_STORIES` 管时代，`AUTHORS` 管人物与作品，人物的 `links` 管脉络关系。新增内容时保持以下原则：

1. 一组节点围绕同一个可比较的问题，而不是简单追求同年。
2. 年代存在争议时，在详情中明确写“约”“成书时代”或“刊行”。
3. 区分作品写成、首次刊行、制度颁布和后世编定。
4. 个人书架只放可进入原典或可靠研究的入口，不追求数量。
5. 区分“直接影响”和“解释性对照”；图谱连线必须用标签说明为什么相连。

## 既有方案预检

开发前查看了 TimelineJS、ChronoZoom 与 Histropedia。它们适合单线叙事或通史浏览，但本项目需要固定的“中国 / 全球”双列、六科交叉、个人收藏与阅读状态，因此采用零依赖本地实现；未来需要大型数据集时，可以再接入 Wikidata 或 TimelineJS 数据格式。
